# NetPractice — Networking Training Project

## Description

NetPractice is an introductory project focused on **computer networking fundamentals**.  
It consists of **10 progressively challenging levels**, each presenting a different network topology.  
The goal is to correctly configure **IP addresses**, **subnet masks**, **routes**, and **default gateways** so that all required machines can communicate.

This project helps you build a solid understanding of:
- **IPv4 addressing**
- **Subnetting** and CIDR notation
- **Default gateways**
- **Static routing**
- **Switching and ARP resolution**
- **Broadcast, network, and host addresses**
- **OSI layers and packet flow**
- **Private vs public IP ranges**

NetPractice provides hands‑on experience with real networking logic, preparing students for more advanced system and network projects.

## Instructions

### Launching the training interface
The project includes a local web interface used to solve the exercises.

To start it:

```bash
./run.sh
```
This command opens a browser window containing all 10 levels.

### Exporting a configuration

After solving a level:
- *Click **Export** in the interface.*
- *Download the generated configuration file.*
- *Rename it if needed (e.g., *level1.txt*, *level2.txt*, etc.).*
- *Place the file **at the root** of your Git repository.*

### Submission requirements

The repository contain exactly **10 exported configuration files**, one for each level:

```bash
level1.txt
level2.txt
...
level10.txt
```
These files are located **at the root of the repository**, next to the README.md.
